Deploy Supercheck on your own infrastructure. Self-hosted mode includes **unlimited features** with all your data staying on your servers.

## Quick Deploy

[![Deploy on Coolify](https://img.shields.io/badge/Deploy%20on-Coolify-6B16ED?style=for-the-badge&logo=coolify&logoColor=white)](https://github.com/supercheck-io/supercheck/blob/main/deploy/coolify/README.md)

Coolify template deployment for Linux hosts. Production still requires running `setup-k3s.sh` on the host so the worker can launch gVisor execution Jobs.

<Callout type="warn">
Self-hosted production uses Docker Compose for the long-lived services and local K3s + gVisor for the execution plane. It is not a pure Docker-only deployment anymore.
</Callout>

## Requirements

| Requirement | Minimum |
|-------------|---------|
| Docker Engine | 20.10+ |
| Linux host | Required for supported production execution |
| Execution plane | Local K3s + gVisor via `setup-k3s.sh` |
| CPU | 4 vCPU |
| Memory | 8GB RAM |
| Storage | 50GB |
| Domain | Optional (required for HTTPS) |

## Services

| Service | Purpose |
|---------|---------|
| **app** | Next.js frontend and API |
| **worker** | Test execution (Playwright/K6) |
| **postgres** | Database |
| **redis** | Job queues |
| **minio** | Artifact storage (S3-compatible) |
| **traefik** | HTTPS reverse proxy (production only) |
